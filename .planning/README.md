# Planning Guide

`.planning/` is the in-repo source of truth for roadmap state, active plans, and execution summaries.

Start here when you need to understand what is complete, what is in progress, or what the next supported GSD step should be.

## Primary Files

- [ROADMAP.md](./ROADMAP.md): phase list and progress
- [STATE.md](./STATE.md): current focus, recent decisions, blockers, and session handoff
- [PROJECT.md](./PROJECT.md): project scope, constraints, and validated requirements

## Phase Artifacts

Each phase directory under `.planning/phases/` contains research, plan, verification, and summary files for that phase. For the current operational cleanup work, use:

- [04-RESEARCH.md](./phases/04-supported-operations-path/04-RESEARCH.md)
- [04-01-PLAN.md](./phases/04-supported-operations-path/04-01-PLAN.md)

## How To Use It

1. Read `STATE.md` to see the current position.
2. Read the active phase plan before making plan-owned changes.
3. Read the matching phase summary after execution to understand what was completed and what was deferred.
