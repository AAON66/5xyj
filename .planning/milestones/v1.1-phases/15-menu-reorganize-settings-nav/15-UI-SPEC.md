## Interaction Contracts

### IC-01: Menu SubMenu Expand/Collapse

- **Trigger:** User clicks a SubMenu group header (常用/数据分析/管理)
- **Behavior:** Group toggles between expanded (showing children) and collapsed (hiding children)
- **Animation:** AntD built-in slide animation (motionDurationMid: 0.2s)
- **Persistence:** Every `onOpenChange` writes current `openKeys` array to `localStorage.setItem('menu-open-keys', JSON.stringify(keys))`
- **Restore:** On mount, `useMenuOpenKeys` reads from localStorage; if absent or parse error, defaults to `['group-common']`
- **Sider collapsed mode:** When Sider collapses, AntD automatically switches SubMenu to popup mode; openKeys state preserved in React state, restored on Sider expand

### IC-02: Menu Item Navigation

- **Trigger:** User clicks a menu item (leaf node)
- **Behavior:** Navigate to route via `react-router-dom`; `selectedKeys` set to `[location.pathname]` (existing pattern)
- **Visual:** Selected item shows `darkItemSelectedBg` + `darkItemSelectedColor` (existing tokens)
- **SubMenu parent:** The parent SubMenu of the selected item stays expanded (AntD handles this automatically when selectedKeys is within an open SubMenu)

### IC-03: Quick Fusion Top Item

- **Position:** First item in sidebar, above all SubMenu groups
- **Visual:** Standard menu item, same height (40px), no grouping wrapper
- **Behavior:** Always visible regardless of which SubMenu groups are open/closed

### IC-04: Settings Page Search

- **Trigger:** User types in search input at top of settings page
- **Debounce:** None (pure client-side filter on small dataset, instant response)
- **Filter logic:** Match `searchTerm` against card `title`, `description`, and `keywords` array (case-insensitive, includes partial match)
- **Match display:** Matching cards remain visible; non-matching cards hidden with no animation (display: none)
- **Highlight:** Matched keyword text wrapped in `<mark>` with `background: colorWarningBg (#FFF7E6 light / AntD dark equivalent)`, `padding: 0`, `borderRadius: 2px`
- **Clear:** Clearing search input restores all cards
- **Empty result:** Show centered empty state: "未找到匹配的设置项" heading + "请尝试其他关键词" body text

### IC-05: Settings Card "Go To" Link

- **Trigger:** User clicks "前往" link on a settings card that has a dedicated page (审计日志, API 密钥, 飞书设置)
- **Behavior:** Navigate to the corresponding route (`/audit-logs`, `/api-keys`, `/feishu-settings`)
- **Visual:** Text link in `colorPrimary (#3370FF)`, no underline, underline on hover

### IC-06: Settings Card Layout

- **Grid:** `Row` + `Col` with `span={8}` (3 columns on desktop), `span={12}` on medium screens, `span={24}` on small screens
- **Gutter:** 24px horizontal, 24px vertical (`[24, 24]`)
- **Card:** AntD `Card` component, uses existing Card token (`paddingLG: 20`, `borderRadiusLG: 8`)
- **Card content:** Title (16px/600) + description (12px/400) + optional "前往" link

### IC-07: Employee Role Menu

- **Condition:** When `userRole === 'employee'`
- **Behavior:** No SubMenu grouping; single flat menu item "员工查询" displayed directly
- **Rationale:** Grouping a single item is visually redundant

## Settings Page Card Order

Cards displayed in this order (top-left to bottom-right, row by row):

1. **外观设置** — 主题模式切换 (inline toggle, no "前往" link)
2. **账号安全** — 修改密码 (inline or "前往" depending on Phase 16 availability)
3. **审计日志** — 查看系统操作记录 ("前往" -> /audit-logs, admin only)
4. **API 密钥** — 管理外部服务密钥 ("前往" -> /api-keys, admin only)
5. **飞书集成** — 飞书同步配置 ("前往" -> /feishu-settings, conditional on feature flag)

Cards filtered by role: admin sees all; hr sees cards 1-2 only.
