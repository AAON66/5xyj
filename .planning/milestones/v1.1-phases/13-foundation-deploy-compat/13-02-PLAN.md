    1. 在组件顶部添加一个 `useRef` 标记用户是否已手动操作主档模式选择：
       ```tsx
       const employeeMasterManualRef = useRef(false);
       ```
       确保 `useRef` 从 React 导入。

    2. 找到 `employeeMasterMode` 的用户触发 set 操作位置（Radio.Group 或 Select 的 onChange），
       在 onChange 回调中添加标记：
       ```tsx
       onChange={(e) => {
         employeeMasterManualRef.current = true;
         setEmployeeMasterMode(e.target.value);
       }}
       ```

    3. 修改获取员工主档数量的 `useEffect` 回调（约 line 192-200），在设置 count 后添加智能默认值逻辑：
       ```tsx
       .then((payload) => {
         if (active) {
           setExistingEmployeeMasterCount(payload.total);
           // D-18: 智能默认值 -- 仅在用户未手动操作时设置
           if (!employeeMasterManualRef.current) {
             if (payload.total > 0) {
               setEmployeeMasterMode('existing');
             }
             // payload.total === 0 时保持初始值 'none'
           }
         }
       })
       ```

    回应 REVIEWS.md MEDIUM 级别关注（竞态条件）：
    使用 useRef（而非 useState）跟踪手动操作，因为 ref 在异步回调中始终读取最新值，
    不受 React 批处理影响。三种场景验证：
    - 服务器有主档 + 用户未操作 -> 默认 existing (OK)
    - 服务器无主档 + 用户未操作 -> 保持 none (OK)
    - 用户已手动选择 none/upload -> API 回调不覆盖 (ref.current = true)
