| Element | Component | Props / Notes |
|---------|-----------|---------------|
| Trigger | Button type="link" danger | "重置密码" text in action column |

### 5. Change Password Modal (shared component)

Used in two contexts: (a) Header dropdown voluntary change, (b) forced change when `mustChangePassword=true`.

| Element | Component | Props / Notes |
|---------|-----------|---------------|

| Old password | Form.Item + Input.Password | label="当前密码", required, prefix={LockOutlined} |

## Interaction States

### Table Row States

| State | Visual | Trigger |
|-------|--------|---------|
| Default | Normal row | - |
| Hover | colorBgLayout background (via Table rowHoverBg token) | Mouse hover |
| Self-row | Switch disabled (grayed), role select disabled in edit modal | `record.id === currentUser.id` |
| Inactive user | Switch unchecked, row text at normal opacity | `is_active === false` |

### Modal Form Validation

| Field | Rule | Error message | Trigger |
|-------|------|---------------|---------|

| Display name | required, max 50 | "请输入显示名" | onBlur + onSubmit |

| Edit user | Modal okButtonProps={{ loading: true }} | OK button only |

### Forced Change Password Flow (D-05)

```
Login -> AuthProvider checks mustChangePassword
  -> If true: render ChangePasswordModal(forced=true) at MainLayout level
  -> Modal cannot be closed (closable=false, maskClosable=false, keyboard=false)
  -> User submits old + new password
  -> On success: update authSession.mustChangePassword=false, call writeAuthSession(), close modal
  -> On failure: show inline error, modal stays open
```
