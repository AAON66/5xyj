## Responsive Breakpoints

| Range | Contract |
|-------|----------|
| `>= 1441px` | Wide desktop. `Sider` expanded to `220px`, breadcrumb visible, page content padding `24px`, inline filters remain visible. |
| `992px - 1440px` | Compact desktop. `Sider` collapses to `64px`, breadcrumb remains visible, content padding `16px`, tables must rely on horizontal scroll instead of column squeeze. |
| `768px - 991px` | Tablet. `Sider` stays collapsed, breadcrumb may remain, filter-heavy pages switch to filter `Drawer`, tables keep horizontal scroll with fixed left identity column. |
| `< 768px` | Mobile. Replace `Sider` with `Drawer`, hide breadcrumb, use compact header, switch filter-heavy pages to `Drawer`, enable workflow sticky action bar, and render employee self-service in card flow. |

Source: Phase 18 discretion on breakpoint alignment, existing `useResponsiveCollapse(1440)`, Ant Design Grid conventions

| Mobile nav | Below `768px`, replace left `Sider` with a left-side `Drawer`. Drawer width is `min(320px, 100vw)`. Use the existing grouped menu structure and existing role filtering. |
