    创建 `frontend/src/hooks/useWebGLSupport.ts`，完全复用 RESEARCH Pattern 2 的实现：

    ```typescript
    import { useEffect, useState } from 'react';

    export type WebGLSupport = 'loading' | 'webgl' | 'fallback';

    export function useWebGLSupport(): WebGLSupport {
      const [support, setSupport] = useState<WebGLSupport>('loading');

      useEffect(() => {
        if (typeof window === 'undefined') {
          setSupport('fallback');
          return;
        }
        try {
          const canvas = document.createElement('canvas');
          const gl =
            (canvas.getContext('webgl2') as WebGLRenderingContext | null) ??
            (canvas.getContext('webgl') as WebGLRenderingContext | null) ??
            (canvas.getContext('experimental-webgl') as WebGLRenderingContext | null);
          if (!gl) {
            setSupport('fallback');
            return;
          }
          // 立即释放 probe context — 避免与 ParticleWave 叠加 2 个活跃 WebGL context（iOS Safari 限 8 个）
          const loseExt = gl.getExtension('WEBGL_lose_context');
          loseExt?.loseContext();
          setSupport('webgl');
        } catch {
          setSupport('fallback');
        }
      }, []);

      return support;
    }
    ```

    关键点：
    - 初始 state 必须是 `'loading'`，不能直接 `'fallback'`，否则 SSR 安全但客户端会闪烁
    - probe context 检测后必须调用 `WEBGL_lose_context.loseContext()` 立即释放（RESEARCH Pitfall 1）
    - `experimental-webgl` 作为第三兜底（IE11 / 老 Firefox）
    - 所有异常路径（try/catch）回退 `'fallback'`，不允许抛到组件外

    不得：
    - 不要写成 `useState(getSupport())` 同步求值（SSR 会爆）
    - 不要保留 probe context，不要把 `gl` 对象返回给调用方
    - 不要用 `@types/three` 里的任何类型（本 hook 不依赖 three）

    本任务对应 D-09「mount 时即检测 WebGL 支持」和 LOGIN-03「优雅降级」的前半段。

      return (

            style={style}

            role="presentation"
            data-testid="css-gradient-background"

            @keyframes cssGradientDrift {

              50%  { background-position: 100% 50%; }
              100% { background-position: 0% 50%; }
