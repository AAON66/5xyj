<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 飞书 OAuth redirect -> callback | 外部 OAuth provider 重定向回系统，携带未验证的 code + state |
| 前端 -> confirm-bind | 前端持有 pending_token + employee_master_id，可被篡改 |
| 前端 -> bind-callback | 已登录用户发起绑定，需验证 JWT + state |
| 前端 -> unbind | 已登录用户发起解绑，需验证 JWT |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | Spoofing | confirm-bind 端点 | mitigate | 使用临时 JWT (5分钟过期, purpose=confirm_bind) 验证 pending_candidates 的 feishu_open_id 来源真实性，拒绝伪造 open_id |
| T-22-02 | Elevation of Privilege | bind-callback | mitigate | bind-callback 端点强制 require_authenticated_user 依赖注入，未认证请求返回 401 |
| T-22-03 | Information Disclosure | pending_candidates | mitigate | 候选列表中 employee_id 脱敏为 `****` + 后4位 (D-06)，不暴露完整工号 |
| T-22-04 | Tampering | confirm-bind employee_master_id | mitigate | 验证 employee_master_id 确实存在于 EmployeeMaster 表，且 open_id 未被其他用户绑定 |
| T-22-05 | Spoofing | OAuth state | mitigate | 复用现有 HMAC-signed state cookie 机制 (已实现)，callback 删除 cookie 防重放 |
| T-22-06 | Denial of Service | feishu_open_id 唯一性冲突 | mitigate | 绑定前显式检查 `db.query(User).filter(User.feishu_open_id == open_id).first()`，冲突返回 409 而非 500 |
| T-22-07 | Elevation of Privilege | 飞书登录角色提升 | mitigate | 新建用户固定 employee 角色 (D-12)，绑定已有用户继承原角色不变 (D-11, D-13) |
</threat_model>
