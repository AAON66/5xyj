## Component Inventory

Phase 17 涉及的 UI 组件及交互规格。

### 1. 多选筛选下拉 (DATA-01)

**组件:** AntD `Select` with `mode="multiple"`

| Property | Value |
|----------|-------|
| mode | "multiple" |
| showSearch | true |
| allowClear | true |
| maxTagCount | 2 (超过 2 个标签时折叠显示 "+N") |
| maxTagPlaceholder | `(omitted) => \`+${omitted.length}...\`` |
| placeholder | 地区: "请选择地区" / 公司: "请选择公司" / 账期: "请选择账期" |
| style.minWidth | 180px |
| style.maxWidth | 320px |

**全选选项:**
- 每个多选下拉的选项列表顶部插入一个"全选"选项
- 全选逻辑: 当全部已选时点击"全选"清空所有；否则选中全部
- "全选"选项使用 `fontWeight: 600` 与普通选项区分

**级联关系 (D-03):**
- 地区选择后 -> 公司下拉只显示已选地区下的公司
- 地区+公司选择后 -> 账期下拉只显示对应的账期
- 选中项被上级筛选排除时自动清除

**URL 持久化:**
- 多选值以逗号分隔存入 searchParams: `region=广州,深圳&company=零一数科,零一裂变`
- 空值时删除对应 param

Source: D-01, D-02, D-03 from CONTEXT.md

### 2. 匹配状态过滤下拉 (DATA-02)

**组件:** AntD `Select` (单选，非多选)

| Property | Value |
|----------|-------|
| mode | default (单选) |
| defaultValue | "matched" |
| allowClear | false |
| style.width | 140px |

**选项:**

| Value | Label | 说明 |
|-------|-------|------|
| "all" | 全部 | 不过滤 |
| "matched" | 已匹配 | match_status = MATCHED |
| "unmatched" | 未匹配 | match_status in (UNMATCHED, DUPLICATE, LOW_CONFIDENCE, MANUAL) |

**位置:** 与地区/公司/账期并列，作为筛选栏的第四个下拉

**URL 持久化:** `matchStatus=matched` (默认值也写入 URL 以保持明确性)

Source: D-05, D-06, D-07 from CONTEXT.md

### 3. 批次删除确认弹窗增强 (DATA-03)

**组件:** AntD `Modal.confirm`

| Property | Value |
|----------|-------|
| title | "确认删除" |
| okText | "确认删除" |
| okType | "danger" |
| cancelText | "取消" |
| icon | ExclamationCircleOutlined (AntD 默认 warning icon) |

**确认内容模板:**

单个批次删除:
```
确认删除批次"${batchName}"吗？

此操作将同时删除：
- ${recordCount} 条明细记录
- ${matchCount} 条匹配结果  
- ${issueCount} 条校验问题

删除后无法恢复。
```

批量删除:
```
确认批量删除 ${count} 个批次吗？

此操作将同时删除所有关联的明细记录、匹配结果和校验问题。

删除后无法恢复。
```

**影响数量获取:** 在弹出确认弹窗前，调用后端 API 预查询关联数据计数。弹窗出现前显示 loading 状态（按钮 loading=true），获取到数据后再弹出弹窗。

Source: D-08, D-09, D-10 from CONTEXT.md

### 4. 筛选栏布局

**整体布局:**

```
[地区 ▼多选] [公司 ▼多选] [账期 ▼多选] [匹配状态 ▼单选]    [搜索框]
```

- 筛选项使用 AntD `Row` + `Col` 布局，或 `Space` with `wrap`
- 各下拉间距: marginXS (8px)
- 搜索框右对齐（保持现有行为）
- 筛选栏在 Card 内部，padding 为 16px

## Interaction States

### 多选下拉

| State | Behavior |
|-------|----------|
| 初始 | 所有下拉为空，显示 placeholder，表格显示全部数据 |
| 选中 1-2 项 | 显示完整 Tag 标签 |
| 选中 3+ 项 | 显示前 2 个 Tag + "+N..." 折叠提示 |
| 清除 | 点击下拉右侧 X 清除全部选中，下级下拉联动重置 |
| 级联加载 | 上级变化时下级选项刷新，已选中但不在新选项中的值自动清除 |
| 全选/取消全选 | 点击"全选"切换全选/取消状态 |

### 匹配状态过滤

| State | Behavior |
|-------|----------|
| 初始 | 默认选中"已匹配" |
| 切换 | 选择新值后立即重新请求数据 |

### 删除确认弹窗

| State | Behavior |
|-------|----------|
| 点击删除 | 按钮进入 loading 状态，预查询关联数据计数 |
| 弹窗出现 | 显示具体影响数量，用户二次确认 |
| 确认删除 | 弹窗 OK 按钮 loading，完成后关闭弹窗、刷新列表、显示 success message |
| 删除失败 | 弹窗关闭，显示 error message |
