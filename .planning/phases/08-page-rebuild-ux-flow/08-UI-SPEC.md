### 2. Responsive Layout (UI-06)

**Source decision:** D-04, D-05

| Breakpoint | Behavior |
|------------|----------|
| > 1440px | Sider expanded (220px), full table columns visible |
| <= 1440px | Sider auto-collapses to icon mode (64px), tables use horizontal scroll |
| <= 1366px | Same as 1440, content area stretches to fill |

**Sider auto-collapse:**
- Use Ant Sider `breakpoint="xl"` prop (maps to 1200px) OR custom `window.matchMedia('(max-width: 1440px)')` listener
- Recommended: custom matchMedia at 1440px to match D-04 exactly
- When auto-collapsed, user can still manually expand via collapse trigger

**Table responsive contract:**
- All Ant Table instances must declare `scroll={{ x: true }}` when column count > 6
- Fixed columns: first column (typically name/employee ID) pinned left with `fixed: 'left'`
- Action column (if present) pinned right with `fixed: 'right'`

**Target resolutions to verify:** 1920x1080, 1440x900, 1366x768

### 3. Chinese Localization (UI-07)

**Source decision:** D-06, D-07

| Layer | Approach |
|-------|----------|
| Ant components | `ConfigProvider locale={zhCN}` in App.tsx root (import zhCN from antd/locale/zh_CN) |
| Business labels | Hardcoded Chinese strings (no i18n framework) |
| API errors | Frontend error interceptor in `services/api.ts` maps HTTP status + backend error codes to Chinese messages |

**Error message mapping file:** `frontend/src/constants/errorMessages.ts`

| HTTP Status | Error Code | Chinese Message |
|-------------|-----------|-----------------|
| 400 | validation_error | 请求参数有误，请检查输入内容 |
| 401 | token_expired | 登录已过期，请重新登录 |
| 401 | invalid_credentials | 用户名或密码错误 |
| 403 | forbidden | 您没有权限执行此操作 |
| 404 | not_found | 请求的资源不存在 |
| 409 | duplicate_entry | 数据已存在，请勿重复提交 |
| 413 | file_too_large | 文件大小超出限制 |
| 422 | unprocessable_entity | 数据格式不正确，请检查后重试 |
| 429 | rate_limited | 操作过于频繁，请稍后再试 |
| 500 | server_error | 服务器内部错误，请稍后重试 |
| 502/503 | service_unavailable | 服务暂时不可用，请稍后重试 |
| timeout | request_timeout | 请求超时，请检查网络连接后重试 |
| network | network_error | 网络连接失败，请检查网络设置 |

Fallback for unmapped codes: "操作失败，请稍后重试（错误码：{code}）"

### 4. Upload-to-Export Workflow Steps (UI-08)

**Source decision:** D-08, D-09

**Steps component spec:**

| Property | Value |
|----------|-------|
| Component | Ant `Steps` |
| Size | `size="small"` |
| Direction | Horizontal |
| Placement | Below page title, above main content area, inside a white Card with 16px vertical padding |
| Clickable | Yes — each step is a link to the corresponding page |

**Step definitions:**

| Step Index | Title | Route | Icon |
|------------|-------|-------|------|
| 0 | 上传文件 | /aggregate | UploadOutlined |
| 1 | 解析处理 | /dashboard | DashboardOutlined |
| 2 | 校验匹配 | /results | CheckCircleOutlined |
| 3 | 导出结果 | /exports | ExportOutlined |

**Step status mapping:**

| Condition | Status | Visual |
|-----------|--------|--------|
| Step completed successfully | `finish` | Green checkmark (#00B42A) |
| Current page matches step | `process` | Blue dot with pulse (#3370FF) |
| Step has warnings/issues | `error` used with warning icon override | Orange exclamation (#FF7D00) |
| Step failed | `error` | Red X (#F54A45) |
| Step not yet reached | `wait` | Gray dot (#8F959E) |

**State detection logic:**
- Step 0 (Upload): `finish` when aggregateSession has completed uploads; `process` when on /aggregate
- Step 1 (Parse): `finish` when parsing complete with no critical errors; `error` (warning) when parsing has warnings; `process` when on /dashboard
- Step 2 (Validate): `finish` when validation/matching complete; `process` when on /results
- Step 3 (Export): `finish` when export downloaded; `process` when on /exports

**Shared component:** Create `frontend/src/components/WorkflowSteps.tsx` — consumed by SimpleAggregate, Dashboard, Results, Exports pages.

## Interaction Contracts

### Sidebar Responsive Behavior

```
Screen width > 1440px:
  Sider expanded (220px) by default
  User can manually collapse/expand via trigger

Screen width <= 1440px:
  Sider collapsed (64px) by default
  User can manually expand (temporarily, until resize)

Collapse animation: Ant Sider built-in slide (motionDurationMid: 0.2s)
```

### Table Scroll Behavior

```
All tables with > 6 columns:
  scroll={{ x: true }}
  Column 1 (name or ID): fixed='left'
  Last column (actions): fixed='right' (if actions exist)

All tables:
  size="small" (global ConfigProvider)
  pagination={{ showSizeChanger: true, showTotal: (total) => `共 ${total} 条` }}
```

### Workflow Steps Click Navigation

```
User clicks a Steps item:
  Navigate to the corresponding route
  Steps component on the target page highlights the correct step

Steps state persists across navigation:
  Read from useAggregateSession hook (upload/parse state)
  Read from page-level state (validate/export state)
```

### Error Interceptor Flow

```
API response arrives with error:
  1. Extract HTTP status code + response body error.code
  2. Look up in errorMessages.ts mapping
  3. If found: display Chinese message via Ant message.error()
  4. If not found: display fallback with error code
  5. For 401 token_expired: also clear session + redirect to /login
```
