## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, table cell padding |
| md | 12px | Ant paddingSM/marginSM -- form field gaps |
| lg | 16px | Default element spacing (Ant padding/margin) |
| xl | 24px | Section padding, content area padding |
| 2xl | 32px | Layout gaps between major sections |
| 3xl | 48px | Page-level vertical spacing |

Exceptions:
- **Anomaly threshold slider row:** Each threshold configuration row uses 8px vertical gap between label and slider. Slider track height is 4px (Ant default).
- **Comparison diff highlight cell:** Inner padding 4px 8px to keep diff cells compact in dense tables.

## Interaction Contracts

### Cross-Period Comparison Flow (INTEL-01)
1. HR navigates to "跨期对比" page
2. HR selects left period and right period from Select dropdowns (populated from API)
3. HR optionally selects region and/or company filters
4. HR clicks "运行对比" (primary button)
5. Loading state: button shows spinner, table shows Skeleton
6. Default view: summary table grouped by company/region, showing aggregated change counts (D-02)
7. HR clicks expand icon on a summary row to see per-employee detail rows (D-02)
8. Detail rows show left value / right value / change for each field, with color coding (D-03)
9. HR can search by name/ID to find specific employees
10. HR can export comparison to Excel

### Anomaly Detection Flow (INTEL-02)
1. HR navigates to "异常检测" page
2. HR selects two periods for comparison
3. HR optionally adjusts thresholds in the collapsible config card (D-06)
4. HR clicks "开始检测" (primary button)
5. Loading state: button shows spinner, "正在检测..." text
6. Results appear in table: one row per anomaly (employee x field combination)
7. HR reviews each anomaly: clicks "确认异常" or "排除（正常变动）" (D-07)
8. HR can batch-select rows and use batch confirm/exclude buttons
9. Status changes persist to database immediately via API (D-08)
10. HR can filter by status to focus on pending items

### Field Mapping Override Flow (INTEL-04)
1. **Entry A (Import result page):** HR views import batch detail, scrolls to "字段映射" section
2. HR sees inline mapping table with current mappings for that batch
3. HR changes a canonical_field via Select dropdown for a row
4. HR clicks "保存修正" on that row
5. System saves mapping, shows `message.success`, records audit log (D-15)
6. Warning banner reminds HR: "映射修正仅影响当前已导入文件" (D-13)

7. **Entry B (Standalone page):** HR navigates to "映射修正" from nav menu
8. HR selects batch and optionally source file
9. Same editing flow as above, but with additional filters (source, confidence)
10. HR can batch-save all draft changes

### Threshold Configuration Interaction (D-06)
1. On AnomalyDetection page, HR clicks to expand threshold config card
2. Each insurance type shows: Chinese label + Slider (0-100%) + InputNumber
3. Default values pre-populated from backend settings
4. Changing slider updates InputNumber and vice versa
5. Thresholds are sent as parameters with the detection request (not persisted client-side)
6. Collapsing the card hides thresholds but retains current values in state

## State Catalog

### PeriodCompare Page States

| State | Condition | Visual Treatment |
|-------|-----------|-----------------|
| Initial | No comparison run yet | Period selectors shown; empty state card below |
| Loading | Comparison API in progress | Button spinner; table area shows Skeleton |
| Summary view | Comparison complete, default | Grouped summary table with expand icons (D-02) |
| Detail expanded | User expanded a summary row | Nested detail table visible with per-employee diffs |
| No results | Comparison complete but no records in either period | Empty state: "所选账期无有效数据" |
| Filtered empty | Search text matches no rows | Empty state: "当前筛选条件下没有可展示的对比记录。" |
| Error | API returned error | Alert type="error" with message |

### AnomalyDetection Page States

| State | Condition | Visual Treatment |
|-------|-----------|-----------------|
| Initial | No detection run yet | Period selectors + threshold config; empty state card |
| Running | Detection API in progress | Button spinner; "正在检测中..." text |
| Results loaded | Detection complete with anomalies | Table with anomaly rows; summary statistics above |
| No anomalies | Detection complete with zero results | Success-tinted Alert: "未发现异常" |
| All processed | All anomalies have been confirmed/excluded | Info Alert: "所有异常均已处理完毕" |
| Filtered empty | Status/field filter yields no results | Empty state with filter-specific message |
| Error | API returned error | Alert type="error" with message |

### MappingManagement Page States

Same as existing `MappingsPage` states (loading, loaded, empty, save-in-progress), plus:

| State | Condition | Visual Treatment |
|-------|-----------|-----------------|
| Draft pending | User changed Select but not yet saved | Row shows unsaved indicator (blue dot or italic text) |
| Batch saving | Batch save in progress | "批量保存" button loading; all row save buttons disabled |

### Inline Mapping Editor States (on ImportBatchDetail)

| State | Condition | Visual Treatment |
|-------|-----------|-----------------|
| Collapsed | Section initially collapsed | Card with collapse icon; "字段映射" title visible |
| Expanded | User expanded section | Compact mapping table visible |

## Accessibility Notes

- All buttons have Chinese aria-labels matching visible text
- Anomaly table: row selection via keyboard (Space to toggle checkbox)
- Expandable rows: Enter key to expand/collapse
- Color is never the sole indicator: all diff cells show numeric values alongside color; all status badges include text
- Threshold sliders: keyboard accessible (arrow keys), linked InputNumber provides precise input
- Anomaly status tags include text labels ("待处理", "已确认", "已排除") alongside color
