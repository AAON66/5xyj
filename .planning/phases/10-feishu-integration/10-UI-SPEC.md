## Interaction Contracts

### Push Flow
1. HR selects sync target from dropdown on FeishuSync page
2. HR clicks "推送到飞书" (primary button)
3. If records with same employee+month exist in Feishu: show Push Conflict Preview modal (D-07)
4. HR chooses overwrite/skip per record or batch action
5. Progress bar appears with NDJSON streaming updates (D-18)
6. On completion: success notification via `message.success`, sync history table auto-refreshes
7. On failure: error notification via `message.error`, failed job appears in history with retry button

### Pull Flow
1. HR selects sync target (must have prior push — D-06)
2. HR clicks "从飞书拉取"
3. System fetches Feishu data, detects conflicts
4. If conflicts: show Pull Conflict Preview modal (D-08)
5. HR selects resolution strategy (D-09)
6. Progress bar during data import
7. On completion: success notification, history refreshes

### Field Mapping Flow
1. Admin navigates to sync target settings
2. Admin clicks "Configure Mapping" on a target
3. FeishuFieldMapping page loads with system fields on left, Feishu columns on right
4. System auto-fetches Feishu column list via API
5. Admin drags from system field handle to Feishu column handle to create edge (mapping)
6. Admin can click "Auto Match" for name-based matching
7. Admin clicks "Save Mapping" to persist

### OAuth Login Flow (D-10, D-12)
1. Login page shows "使用飞书登录" button (only when feature flag on)
2. Click redirects to Feishu authorization page
3. User authorizes on Feishu
4. Redirect back to frontend callback URL
5. Frontend sends code to backend
6. Backend issues system JWT
7. Frontend stores JWT and redirects to role-appropriate workspace

### Credential Not Configured State
1. FeishuSync page detects no credentials configured
2. Shows `Alert type="info"` with guidance: "飞书应用凭证未配置，请联系管理员在飞书设置中配置 App ID 和 App Secret。"
3. Push/Pull buttons disabled with tooltip "请先配置飞书凭证"

## State Catalog

### FeishuSync Page States

| State | Condition | Visual Treatment |
|-------|-----------|-----------------|
| Feature disabled | FEISHU_SYNC_ENABLED=false | Route redirects to workspace; nav item hidden |
| No credentials | Feature on but app_id/secret missing | Info Alert with setup guidance; buttons disabled |
| No sync targets | Credentials OK but no targets configured | Empty state with link to settings |
| Idle | Targets configured, no active sync | Target selector + push/pull buttons enabled |
| Syncing | Active push or pull in progress | Progress bar visible; buttons disabled; streaming updates |
| Sync complete | Job finished successfully | Success message; history table refreshed |
| Sync failed | Job failed | Error message; retry button on failed job row |
| Push conflict | Existing records detected during push | Push Conflict Preview modal open |
| Pull conflict | Differences detected during pull | Pull Conflict Preview modal open |

### FeishuSettings Page States

| State | Condition | Visual Treatment |
|-------|-----------|-----------------|
| No credentials | First-time setup | Empty credential form with info guidance |
| Credentials saved | App ID/Secret configured | Form shows masked values; "Update" button |
| Loading targets | Fetching sync config list | Table skeleton loading |
| Targets listed | Configs exist | Table with edit/delete actions |
| Adding target | Drawer open for new config | Drawer with form fields |
| Fetching fields | Loading Feishu columns for a target | Spin indicator on fetch button |

### FeishuFieldMapping Page States

| State | Condition | Visual Treatment |
|-------|-----------|-----------------|
| Loading | Fetching system fields + Feishu columns | Centered Spin on canvas |
| No Feishu columns | Feishu field fetch failed or empty table | Error Alert with retry option |
| Empty mapping | Fields loaded, no connections yet | Nodes visible, no edges; guidance text below canvas |
| Mapping in progress | Some edges created | Canvas shows connected nodes; save button enabled |
| Saving | Persisting mapping | Save button loading state |
| Saved | Mapping persisted | Success message |

## Accessibility Notes

- All buttons have Chinese aria-labels matching visible text
- Field mapping canvas: keyboard support via @xyflow/react built-in (Tab to focus nodes, Enter to connect)
- Conflict tables: row expansion via keyboard (Enter on expandable row)
- Color is never the sole indicator of status — status badges include text labels alongside color
- Progress bar includes aria-valuenow for screen readers
