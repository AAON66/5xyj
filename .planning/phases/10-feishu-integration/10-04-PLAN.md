`handleFeishuLogin` function:
```typescript
async function handleFeishuLogin() {
  try {
    const url = await fetchFeishuAuthorizeUrl();
    // NOTE: The backend's /authorize-url endpoint sets a signed httpOnly cookie
    // with the OAuth state for CSRF validation (addresses review H2).
    // The cookie is automatically sent with the callback request.
    window.location.href = url;
  } catch {
    message.error('获取飞书授权链接失败');
  }
}
```

Also add a callback handler. Check if current URL has `?code=` and `?state=` params (Feishu redirect back). If so, call `feishuOAuthCallback(code, state)`. The backend validates the `state` against the signed cookie set during `/authorize-url` (the cookie is automatically included in the POST request since it is same-origin). Store the returned JWT using the auth session (same as password login), and redirect to workspace.

```typescript
useEffect(() => {
  const params = new URLSearchParams(window.location.search);
  const code = params.get('code');
  const state = params.get('state');
  if (code && state && feishu_oauth_enabled) {
    feishuOAuthCallback(code, state)
      .then((result) => {
        // Store token using the same auth session mechanism
        saveAuthSession({ token: result.access_token, role: result.role as AuthRole, username: result.username });
        window.location.href = '/';
      })
      .catch(() => message.error('飞书登录失败'));
  }
}, []);
```

Import `saveAuthSession` from the auth session module, `Divider` and `ApiOutlined` from antd/icons.

IMPORTANT: The `feishuOAuthCallback` function should use the existing `apiClient` (which sends cookies automatically for same-origin requests). The backend reads the `feishu_oauth_state` httpOnly cookie to validate the state parameter server-side (H2). The frontend does NOT need to manually manage the state -- it just passes `code` and `state` from the URL to the backend, and the cookie handles CSRF validation.
