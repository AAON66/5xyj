  Build SQLAlchemy query on NormalizedRecord with optional `.filter()` for each param.
  **Default sort: `NormalizedRecord.created_at.desc(), NormalizedRecord.id.asc()`** for deterministic pagination.
  Use `.count()` subquery for total (NOT `len(query.all())`).
  Use `.offset(page * page_size).limit(page_size)` for items.
  Convert Decimal to float via schema.

- `get_filter_options(db, region=None, company_name=None) -> FilterOptionsRead` **(addresses review concern #3 -- cascading filter contract)**:
  This endpoint implements the cascading constraint for D-02:
  - `regions`: Always query ALL DISTINCT region values from NormalizedRecord, sorted alphabetically. No filter applied -- regions are the top-level cascade.
  - `companies`: If `region` param is provided, query `DISTINCT company_name WHERE region = :region`. If region is None, query ALL DISTINCT company_name. Sorted alphabetically.
  - `periods`: If both `region` AND `company_name` are provided, query `DISTINCT billing_period WHERE region = :region AND company_name = :company_name`. If only `region`, query `DISTINCT billing_period WHERE region = :region`. If neither, query ALL DISTINCT billing_period. Sorted descending (newest first).
  - Filter out NULL values from all three lists.

  GROUP BY employee_id, person_name, company_name, region.

  Apply optional filters.
  **Default sort: `person_name ASC NULLS LAST, employee_id ASC NULLS LAST`** for deterministic pagination.
  Use subquery for total count of groups (wrap grouped query in a subquery, then count).
  All func.sum() results: convert to float with `float(val or 0)`.

**Duplicate detection (addresses review concern #1):**
Use `id_number + billing_period` as the PRIMARY duplicate detection key (not person_name), because id_number is unique per person while person_name can have collisions. For records where id_number is NULL, fall back to `person_name + company_name + billing_period` as a secondary detection key. This is cross-batch (not limited to a single batch), because a person truly imported twice across batches is still a duplicate.

Implementation:
```python
def _count_duplicates_for_batch(db: Session, batch_id: str) -> int:
    """Count duplicate records within a batch.

    Primary key: id_number + billing_period (cross-batch, most reliable).
    Fallback: person_name + company_name + billing_period (for records missing id_number).
    """
    from sqlalchemy import and_, literal

    # Primary: id_number + billing_period duplicates (only records that HAVE id_number)
    primary_dupes = (
        db.query(func.count(literal(1)))
        .filter(
            NormalizedRecord.batch_id == batch_id,
            NormalizedRecord.id_number.isnot(None),
            NormalizedRecord.billing_period.isnot(None),
        )
        .filter(
            # This record's id_number+billing_period appears more than once across ALL batches
            NormalizedRecord.id_number.in_(
                db.query(NormalizedRecord.id_number)
                .filter(
                    NormalizedRecord.billing_period == NormalizedRecord.billing_period,
                    NormalizedRecord.id_number.isnot(None),
                )
                .group_by(NormalizedRecord.id_number, NormalizedRecord.billing_period)
                .having(func.count(NormalizedRecord.id) > 1)
                .correlate(None)
            )
        )
        .scalar() or 0
    )

    # Fallback: person_name + company_name + billing_period for records WITHOUT id_number
    fallback_dupes = (
        db.query(func.count(literal(1)))
        .filter(
            NormalizedRecord.batch_id == batch_id,
            NormalizedRecord.id_number.is_(None),
            NormalizedRecord.person_name.isnot(None),
            NormalizedRecord.billing_period.isnot(None),
            NormalizedRecord.company_name.isnot(None),
        )
        .filter(
            # Check if this name+company+period combo exists more than once
            NormalizedRecord.person_name.in_(
                db.query(NormalizedRecord.person_name)
                .filter(
                    NormalizedRecord.id_number.is_(None),
                    NormalizedRecord.person_name.isnot(None),
                    NormalizedRecord.company_name.isnot(None),
                    NormalizedRecord.billing_period.isnot(None),
                )
                .group_by(
                    NormalizedRecord.person_name,
                    NormalizedRecord.company_name,
                    NormalizedRecord.billing_period,
                )
                .having(func.count(NormalizedRecord.id) > 1)
                .correlate(None)
            )
        )
        .scalar() or 0
    )

    return primary_dupes + fallback_dupes
```

NOTE: The above is pseudocode showing the logic. The actual implementation should use a simpler approach:
- For primary duplicates: query NormalizedRecord grouped by (id_number, billing_period) WHERE id_number IS NOT NULL, HAVING count > 1. For each such group, count how many records belong to the target batch_id. Sum those counts.
- For fallback duplicates: same pattern but grouped by (person_name, company_name, billing_period) WHERE id_number IS NULL AND person_name IS NOT NULL.
- The executor should implement whichever SQL approach is cleanest. The key constraint is: use id_number as primary key, person_name+company_name as fallback.

For each batch (query recent ImportBatch ordered by created_at DESC, limit):
- `missing_fields`: count NormalizedRecord where batch_id matches AND (person_name IS NULL OR id_number IS NULL OR employee_id IS NULL)
- `anomalous_amounts`: count NormalizedRecord where batch_id matches AND payment_base IS NOT NULL AND (payment_base < ANOMALY_THRESHOLDS["payment_base_min"] OR payment_base > ANOMALY_THRESHOLDS["payment_base_max"])
- `duplicate_records`: use the dual-key detection described above
- `record_count`: total NormalizedRecord count for the batch
Sum all batch-level counts for totals.
